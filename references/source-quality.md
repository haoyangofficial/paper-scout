# Source Quality and Ranking

Use these heuristics when deciding which papers to include and how prominently to rank them.

## Publication Status

Rank strongest to weakest:

1. Published journal article in a reputable venue.
2. Main-track peer-reviewed conference paper.
3. Workshop paper, short paper, demo paper, or findings paper.
4. Accepted paper with clear proceedings page but no final PDF yet.
5. Preprint with strong relevance, citations, known authors, or code.
6. Preprint with no code, no citations, and no independent signals.

Do not discard preprints automatically. In fast-moving areas, preprints may be essential, but label them clearly.

## Code Availability

Rank strongest to weakest:

1. Official implementation linked from the paper, author page, Papers with Code, or publisher page.
2. Repository owned by a paper author, lab, or project organization.
3. Community implementation with active maintenance and clear paper mapping.
4. Unverified repository found only by title search.

Download a repository zip when it is likely relevant. Record uncertainty instead of overstating provenance.

## Open Access

Prefer open PDFs from publisher, conference proceedings, arXiv, PubMed Central, institutional pages, or Semantic Scholar `openAccessPdf`. Avoid suspicious PDF mirrors when official or archival sources exist.

## Selection Balance

When many candidates are available, choose a balanced set:

- 60-80% published journal/conference papers.
- 20-40% high-value preprints only when they add novelty, recency, code, or coverage.
- Include methodological diversity rather than near-duplicates from the same group unless the user asked for a narrow subthread.

## Report Caveats

State the search date, search terms, time window, APIs or sites used, and known blind spots. If the search is limited by API rate limits, missing abstracts, blocked PDFs, or unavailable code archives, record that in the final Markdown.
